01
專案簡介


這是一個公司行銷電子報系統後台的改版專案，​整合各部門對於系統的需求和管理，提升電子報系統效率。我的目標是透過存在系統已久的問題和訪談使用該系統的各部門同事，思考再次優化後台系統。

02
研究問題與目標


有自己的行銷電子報系統但效率低和電郵成效無法追蹤

公司提供的派送電子報系統龐大，但後期發展各項業務後，業務使用的客戶名單和原本系統的名單各自為政。導致每個行銷活動的電子報成效無法有效的追蹤，以及當客戶群重複收信後，演變我們成為擋信名單。

各部門使用系統的同仁角色眾多需求各個不同 當各部門同仁使用該系統時，會有以下問題

設計排信

每份電子報都需要編寫語法，不知道可不可以建置可拖曳的元件快速建立電子報？而且排信系統介面對於新人不友善，操作好複雜。

行銷數據

每個活動的成效不知道好不好？希望能有開信率，點閱率，退信率等數據。

主管簽核

每個活動都要順簽才通過，還要口頭通知下一位主管，不知道能不能客製每個活動的簽核主管呢？

03
設計目標


  • 1. 透過訪談重點同仁（設計排信人員），在第一階段重新規劃給新手和熟手友善且幫助快速建立電子報和排信的介面設計。

  • 2. 在系統首頁提供給各部門同仁重點資訊，方便各部門同仁登入後查看。

  • 3. 整合舊系統和業務資料庫客戶名單，當資料來源整合後，新系統使用時只需同步更新成最新的資料庫即可。

06
User Interface Proposal


#1 首頁提供最新即時的訊息

在首頁的介面上，提取各單位在意的即時資訊，如果需要更進一步了解細節，可以選擇在觀看更多。

  #2 表單欄位提供空間

在傳統的表單上，提供的是觀看空間較窄小的欄位，在改版上提供欄位舒適的空間感。

  #3 表單欄位的提示訊息

在傳統的表單上，每個訊息欄位沒有說明，讓新手使用上難以理解該欄位需要輸入什麼資訊。
在改版上提供需要的提示訊息。

  #4 系統出現錯誤的解決路線

在傳統的系統畫面上，當出現操作錯誤產生當機時，提供用戶多方解決路線，避免用戶產生困惑。
